Step by step

實作步驟

步驟 1｜分析你的最佳發文時段

查看各平台的 Insights / Analytics，找出你的受眾最活躍的時段。沒有數據的話，先用通用建議：Facebook 早上 9–11 點、Instagram 晚上 7–9 點、LinkedIn 週二三上午。

步驟 2｜安裝排程相關 Skills

```bash
npx clawhub@latest install social-scheduler
npx clawhub@latest install facebook-publisher
npx clawhub@latest install instagram-publisher
npx clawhub@latest install twitter-publisher
```

步驟 3｜建立內容儲存庫

在 Google Sheet 或 Notion 建立一個表格，欄位包含：貼文內容、目標平台、排程時間、配圖路徑、hashtag、狀態（待發/已發/失敗）。

步驟 4｜批次匯入內容

將你已經準備好的貼文（可搭配教案 06「一文多平台」流程產出）批次匯入儲存庫。建議一次準備至少一週份的內容。

步驟 5｜設定排程規則

配置排程邏輯：
- 每日固定時段發文（如每天早上 9 點 + 晚上 7 點）
- 依平台設定不同的發布時間
- 設定「不發文日」（如國定假日）

步驟 6｜串接平台 API

將各社群平台的 API Token 設定好。注意：Instagram 需要透過 Facebook Graph API 發布；Twitter 需要 Developer Account。

步驟 7｜設定發布前預覽

每天排定發布的貼文，提前 1 小時推送到你的 Telegram，讓你有機會最後確認或取消。

步驟 8｜監控與補發機制

設定發布失敗通知。常見失敗原因：API Token 過期、圖片格式不支援、文字超過字數限制。失敗的貼文自動加入「待補發」佇列。

常見錯誤

  • **API Token 過期沒更新**：大部分社群平台的 Token 有效期限是 60–90 天。設定到期提醒。
  • **忽略平台政策**：Instagram 不允許第三方直接發 Reels；Twitter 的 API 分免費和付費版，功能不同。先確認政策再設定。
  • **排程太密集**：每天發 5 篇不如每天 1–2 篇高品質內容。過度發文會被演算法懲罰。
  • **不追蹤成效**：排了就忘，不看數據。每週花 15 分鐘看哪些貼文表現好，調整內容策略。

升級方向

  • 智慧排程：AI 根據歷史互動數據自動選擇最佳發布時間
  • 自動回覆留言：發文後自動回覆前 10 則留言，提高互動率
  • 內容日曆視覺化：用日曆介面管理所有排程，一眼看到本月的發文規劃
  • 跨帳號管理：同時管理多個品牌的社群帳號
