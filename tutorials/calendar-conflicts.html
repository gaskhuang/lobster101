用 AI 自動排解行事曆衝突——再也不用手動搬會議

當三場會議撞在同一個時段，AI 幫你根據優先順序自動重新排程，還會通知所有參與者。

這篇教案適合誰

行事曆經常被會議塞滿、時段衝突頻繁的主管或 PM同時管理多個專案、需要在不同團隊間切換的人討厭手動拖拉行事曆、跟人喬時間的所有忙碌工作者

行事曆衝突是忙碌工作者的日常噩夢。手動排解衝突要考慮：哪些會議可以移、每個人的空閒時段、會議室可用性、時區差異。你需要 AI 幫你自動偵測衝突、根據優先順序建議調整方案、並通知相關人員。

Step by step

實作步驟

步驟 1｜連接 Google Calendar

設定 Google Calendar API 存取權限。建立 OAuth 認證，讓 Agent 能讀取和修改你的行事曆。

步驟 2｜安裝排解相關 Skills

```bash
npx clawhub@latest install calendar-connector
npx clawhub@latest install conflict-resolver
npx clawhub@latest install email-notifier
```

步驟 3｜定義會議優先順序規則

設定你的優先順序策略：
- 客戶會議 > 內部會議 > 1:1 > 例行會議
- 有外部參與者的會議不輕易移動
- 超過 5 人的會議不輕易移動
- 你的「專注時間」區塊不能被覆蓋

步驟 4｜設定衝突偵測排程

配置每日檢查頻率。建議每天早上 7 點和下午 3 點各檢查一次：早上處理當天衝突，下午預處理隔天衝突。

步驟 5｜配置自動排解邏輯

設定 AI 排解衝突的行為：
- 優先度低的會議自動搜尋同一天的空閒時段
- 找不到同天空閒則推遲到下一個工作天
- 產出建議方案後先通知你確認，再通知其他人

步驟 6｜設定通知模板

當會議需要調整時，自動發送通知給相關參與者。通知內容包含：原始時間、建議新時間、調整原因。語氣要禮貌專業。

步驟 7｜處理例外情況

設定「不可移動」標記。某些會議（如面試、客戶 Demo）絕對不能動，用特定標籤或行事曆顏色標記。

步驟 8｜每週檢討與優化

每週五回顧本週的衝突排解紀錄：哪些排解方案被你駁回了？為什麼？據此調整優先順序規則。

常見錯誤

  • **沒設「不可移動」標記**：重要會議被 AI 自作主張移走，比衝突還麻煩。一定要設白名單。
  • **完全自動化太早**：先用「建議模式」（AI 建議你確認後才執行），確認邏輯正確後再開啟自動執行。
  • **忽略時區**：跨時區團隊的會議排解要考慮每個人的工作時間，不要把會議排到別人的半夜。
  • **通知不夠清楚**：只說「會議已改時間」不夠，要說明原因和新時間，否則參與者會困惑。

升級方向

  • 跨日曆同步：同步多個 Google Calendar（個人 + 工作 + 共用日曆）
  • 智慧會議時長：AI 分析你的會議模式，建議縮短不必要的 1 小時會議為 30 分鐘
  • 自動空出專注時間：每天自動保留 2 小時的無會議區塊
  • 會議負荷預警：當下週的會議數超過閾值時提前警告
