Step by step

實作步驟

步驟 1｜建立你的內容風格指南

定義各平台的品牌語氣：部落格（專業深度）、Facebook（親切口語）、Instagram（視覺搭配簡短文案）、LinkedIn（商務專業）、Twitter/X（精簡有力）。這份指南會成為 AI 改寫的依據。

步驟 2｜安裝內容複製 Skills

```bash
npx clawhub@latest install content-repurposer
npx clawhub@latest install social-publisher
npx clawhub@latest install image-generator
```

步驟 3｜撰寫原始長文

用 Markdown 撰寫一篇完整的部落格文章（800–1500 字）。這是你的「母內容」，所有平台版本都從這裡衍生。

步驟 4｜設定各平台改寫規則

為每個平台建立改寫 Prompt：
- Facebook：300 字以內、口語化、加問句引發互動、附 3–5 個 hashtag
- Instagram：150 字以內、加 emoji、附 15–20 個 hashtag
- LinkedIn：500 字以內、專業語氣、附 3 個 hashtag
- Twitter/X：280 字以內、分成 3–5 則串文、附 2–3 個 hashtag
- 電子報：保留完整內容但加入 CTA 和個人化開頭

步驟 5｜配置圖片生成（選用）

為 Instagram 和 Facebook 貼文自動生成配圖或文字圖卡。設定品牌色系、字型、Logo 位置。

步驟 6｜預覽與人工審核

一次產出所有平台版本後，逐一預覽。AI 改寫偶爾會偏離品牌調性，花 5 分鐘快速修改比事後下架好。

步驟 7｜設定發布流程

選擇手動發布（AI 產好內容你再貼）或自動發布（直接串接各平台 API）。建議初期先手動，確認品質穩定後再自動化。

步驟 8｜建立內容行事曆

設定每週的內容生產流程：週一寫母內容 → AI 產出多平台版本 → 週二至週五分別發布。用 Google Sheet 或 Notion 追蹤發布狀態。

常見錯誤

  • **改寫品質沒把關就直接發**：AI 改寫可能產出不自然的語句或錯誤的 hashtag。務必人工審核。
  • **所有平台用同一種語氣**：LinkedIn 不能太口語，Instagram 不能太長。一定要針對平台調整。
  • **忽略平台演算法差異**：Facebook 喜歡長文互動、Instagram 靠 hashtag 觸及、Twitter 靠串文增加曝光。了解規則才能最大化效果。
  • **母內容品質不佳**：AI 改寫再厲害也救不了爛文章。花時間把原始內容寫好是最重要的。

升級方向

  • A/B 測試：同一主題產出 2–3 種不同風格的版本，測試哪種互動率最高
  • 數據回饋迴圈：追蹤每篇貼文的互動數據，自動調整改寫策略
  • 多語系內容：一篇中文文章自動翻譯成英文、日文版本
  • 影片腳本：從文章自動產出 YouTube 短影片或 Reels 腳本
