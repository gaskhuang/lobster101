這篇教案適合誰

每天花超過 1 小時瀏覽產業新聞的創業者、行銷主管、顧問想掌握市場動態但時間有限的中小企業老闆需要定期彙報產業趨勢的研究人員或幕僚

手動追蹤多個新聞來源（RSS、電子報、社群）既耗時又容易遺漏關鍵資訊。你需要一個每日自動幫你彙整、摘要、分類的流程，省下每天 1–2 小時的閱讀時間。

Step by step

實作步驟

步驟 1｜定義你的情報來源

列出 5–10 個你每天必看的資訊來源：產業新聞網站、競品 Blog、Twitter/X 帳號、Reddit 子版。把它們整理成一張清單，包含 RSS 連結或網址。

步驟 2｜安裝所需 Skills

```bash
npx clawhub@latest install rss-reader
npx clawhub@latest install web-scraper
npx clawhub@latest install ai-summarizer
npx clawhub@latest install telegram-notifier
```

步驟 3｜設定 RSS 抓取任務

建立一個 `sources.yaml` 設定檔，將你的來源 URL 逐一填入。設定抓取頻率為每日一次，時間設在早上 7:30。

步驟 4｜配置摘要模板

撰寫 Prompt 模板，告訴 AI 你要的摘要格式：「列出前 10 則最重要新聞，每則用一句話摘要，附上原始連結，最後給出 3 個行動建議。」

步驟 5｜設定分類邏輯

依照你關心的面向（競品動態、法規變化、技術趨勢、市場數據）設定自動分類標籤，讓摘要報告一目了然。

步驟 6｜串接推送管道

將產出的摘要接上 Telegram Bot 或 Email，讓報告每天早上 8 點自動送到你手上。

步驟 7｜設定排程自動化

用 cron job 或 n8n 將整條流程串起來，設定每日定時觸發。確認排程正確執行。

步驟 8｜測試與微調

手動觸發一次，檢查摘要品質。調整 Prompt 中的摘要長度、分類標籤、推送格式，直到符合你的閱讀習慣。

常見錯誤

  • **來源太多太雜**：一開始不要超過 10 個來源，否則摘要會又長又沒重點。先聚焦核心來源，再逐步擴充。
  • **Prompt 太模糊**：「幫我摘要」不夠好，要明確指定格式、長度、分類方式。
  • **忽略 RSS 格式差異**：不同網站的 RSS 結構不同，部分網站可能需要用 Web Scraper 取代 RSS Reader。
  • **沒設錯誤通知**：排程失敗時你不會知道。務必加上失敗通知機制。

升級方向

  • 加入競品定價監控：自動偵測競品價格或功能變動
  • 串接 Notion / Google Sheet：將每日摘要自動歸檔，累積成可搜尋的知識庫
  • 多語言情報：抓取英文、日文來源並自動翻譯成中文
  • 週報自動產出：每週五自動彙整當週摘要，產出一份週報
