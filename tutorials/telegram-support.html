Step by step

實作步驟

步驟 1｜建立 Telegram Bot

在 Telegram 找 @BotFather，輸入 `/newbot` 建立一個新的 Bot。取得 Bot Token 並妥善保存。設定 Bot 的名稱與描述。

步驟 2｜整理你的 FAQ 知識庫

把客戶最常問的 30–50 個問題整理成結構化文件（Markdown 或 JSON）。每個問答對包含：問題、標準回答、相關連結。

步驟 3｜安裝相關 Skills

```bash
npx clawhub@latest install telegram-bot
npx clawhub@latest install rag-engine
npx clawhub@latest install knowledge-base
```

步驟 4｜建立 RAG 知識庫索引

將你的 FAQ 文件餵入 RAG Engine，建立向量索引。這讓 AI 能夠根據客戶的「自然語言提問」找到最相關的答案，而不只是關鍵字比對。

步驟 5｜設定回覆邏輯

配置 AI 的回覆策略：
- 信心度 > 85%：直接回覆答案
- 信心度 60–85%：回覆答案但附上「如需進一步協助，請輸入 /human」
- 信心度 < 60%：自動轉接真人客服

步驟 6｜設計回覆模板

為不同情境設計回覆語氣與格式。例如：詢價用正式語氣 + 附價目表連結；售後問題用親切語氣 + 附操作教學連結。

步驟 7｜串接人工轉接

當 AI 無法回答或客戶主動要求時，自動通知真人客服接手。設定通知管道（另一個 Telegram 群組或 Email）。

步驟 8｜上線測試與監控

先在內部測試群組模擬 20 種不同問法，確認回覆品質。上線後持續監控「未命中」的問題，每週更新知識庫。

常見錯誤

  • **知識庫太少**：只放 10 個 FAQ 不夠。客戶問法多變，至少要 30 個以上的問答對，且每個問題要有 2–3 種不同問法。
  • **沒設轉接機制**：AI 回答不了卻沒通知真人，客戶會感覺被忽略，反而更差。
  • **語氣太機器人**：回覆模板要有品牌語氣，不要像制式回覆。加入 emoji、稱呼客戶名字。
  • **忽略持續優化**：上線後第一週一定會有大量 AI 答不好的問題。每天花 10 分鐘看未命中紀錄，持續更新知識庫。

升級方向

  • 多語系客服：自動偵測客戶語言，用對應語言回覆
  • 訂單查詢整合：串接你的訂單系統，客戶可以直接用 Bot 查物流狀態
  • 主動推播：定期推送促銷、新品資訊給曾互動過的客戶
  • 數據分析儀表板：統計最常被問的問題、回覆滿意度、轉接率
