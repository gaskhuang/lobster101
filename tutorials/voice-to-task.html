這篇教案適合誰

常在移動中產生想法但來不及記錄的業務、顧問、創業者開會後總忘記 action items 的團隊負責人習慣用語音備忘但事後懶得整理的自由工作者

語音筆記通常錄完就被遺忘。手動聽錄音、抄筆記、拆任務太花時間。你需要一條自動化流程：錄音 → 轉文字 → 抽取任務 → 寫入待辦工具。

Step by step

實作步驟

步驟 1｜選擇錄音工具

在手機上選一個可以自動同步錄音檔的 App。iOS 用「語音備忘錄」搭配 iCloud，Android 用「Google 錄音工具」搭配 Google Drive。確保錄音檔能自動同步到電腦或雲端。

步驟 2｜安裝語音轉文字 Skill

```bash
npx clawhub@latest install whisper-transcriber
npx clawhub@latest install task-extractor
npx clawhub@latest install todoist-connector
```

步驟 3｜設定監聽資料夾

指定一個資料夾（如 `~/VoiceNotes/`），讓 Agent 自動偵測新增的音檔。一有新錄音就觸發轉錄流程。

步驟 4｜配置轉錄參數

設定語言（繁體中文）、模型大小（建議 medium 以上以確保中文準確度）、輸出格式（純文字 + 時間戳記）。

步驟 5｜撰寫任務抽取 Prompt

告訴 AI：「從以下逐字稿中，抽取所有可執行的任務。每個任務需包含：任務描述、優先度（高/中/低）、建議截止日期。忽略閒聊和重複內容。」

步驟 6｜串接待辦工具

將抽取出的任務自動寫入你常用的工具：Todoist、Notion Database、或 Trello 看板。設定對應的欄位映射。

步驟 7｜加入確認機制

在自動寫入前，先將任務清單推送到 Telegram 讓你確認。你回覆「OK」後才正式寫入待辦工具，避免 AI 誤判。

步驟 8｜建立日常習慣

設定每天固定時段（如晚上 9 點）自動處理當天所有語音筆記，隔天早上你就能看到整理好的任務清單。

常見錯誤

  • **錄音品質太差**：在嘈雜環境錄音會嚴重影響轉錄準確度。盡量在安靜處錄，或使用降噪麥克風。
  • **語音太隨性**：完全無結構的意識流很難抽取任務。錄音時稍微有意識地說「任務一⋯⋯任務二⋯⋯」會大幅提升效果。
  • **一次處理太多檔案**：長音檔（超過 30 分鐘）建議先切段再處理，避免記憶體不足或摘要品質下降。
  • **沒有人工確認就直接寫入**：AI 可能誤判優先度或截止日，務必保留確認步驟。

升級方向

  • 會議錄音自動分派：辨識不同發言人，將任務自動分派給對應的人
  • 多語言支援：英文會議也能轉錄並抽取中文任務
  • 語音日記：除了任務，也自動整理成每日工作日誌
  • 串接行事曆：有截止日的任務自動建立日曆事件
