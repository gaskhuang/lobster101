Step by step

實作步驟

步驟 1｜選擇錄音方式

線上會議：開啟 Google Meet / Zoom 內建錄音功能，或用 OBS 錄製系統音訊。實體會議：用手機或錄音筆放在桌上。確保錄音品質清楚。

步驟 2｜安裝會議紀錄 Skills

```bash
npx clawhub@latest install whisper-transcriber
npx clawhub@latest install meeting-summarizer
npx clawhub@latest install task-extractor
npx clawhub@latest install notion-connector
```

步驟 3｜設定逐字稿轉錄

配置語言（中文/英文/混合）、模型大小、說話人辨識（diarization）。說話人辨識能區分「誰說了什麼」，對會議紀錄至關重要。

步驟 4｜設計會議紀錄模板

定義你要的紀錄格式：
- 會議基本資訊（日期、參與者、時長）
- 討論重點（按議題分段）
- 決議事項
- 行動項目（負責人、截止日、狀態）

步驟 5｜配置行動項目抽取

撰寫 Prompt：「從會議紀錄中抽取所有行動項目。每項包含：任務描述、負責人（根據逐字稿中的發言判斷）、建議截止日、優先度。」

步驟 6｜串接筆記工具

將會議紀錄自動寫入 Notion Database 或 Google Docs。行動項目同步寫入 Todoist 或 Trello，並指定負責人。

步驟 7｜建立追蹤機制

設定每日提醒：在 Telegram 或 Slack 推送即將到期的 action items。每週五自動產出「本週未完成項目清單」。

步驟 8｜團隊上線與回饋

讓團隊試用 2 週，收集回饋。常見需調整的地方：紀錄格式、行動項目的粒度、提醒頻率。持續迭代。

常見錯誤

  • **錄音品質差**：線上會議如果收音不好，轉錄準確度會暴跌。建議參與者用耳機麥克風，減少迴音。
  • **沒做說話人辨識**：不知道誰說了什麼，行動項目就無法正確分派。這步不能省。
  • **行動項目太模糊**：「John 跟進這件事」不是好的 action item。要具體到「John 在 3/15 前提交客戶提案初稿」。
  • **紀錄沒人看**：再好的紀錄放在沒人打開的文件裡也沒用。要主動推送到團隊日常使用的頻道。

升級方向

  • 即時紀錄：會議進行中就即時產出紀錄，不用等會後
  • 跨會議追蹤：自動追蹤同一個 action item 在多次會議中的進展
  • 會議效率分析：統計每場會議的發言比例、決議效率、action item 完成率
  • 自動排下次會議：當 action items 全部完成或到期時，自動安排覆盤會議
