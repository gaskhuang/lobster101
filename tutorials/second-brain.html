你的知識分散在太多地方：Notion 筆記、Google Docs、瀏覽器書籤、PDF 文件、Slack 對話。傳統搜尋只能比對關鍵字，找不到「語意相關」的內容。你需要一個統一的知識庫，能用自然語言問問題並得到精準答案。

Step by step

實作步驟

步驟 1｜盤點你的知識來源

列出你目前所有的知識存放位置：Notion workspace、Google Drive 資料夾、本地 Markdown 檔案、常用網站書籤、PDF 電子書。按優先度排序。

步驟 2｜安裝知識庫 Skills

```bash
npx clawhub@latest install rag-engine
npx clawhub@latest install doc-loader
npx clawhub@latest install notion-connector
npx clawhub@latest install pdf-parser
```

步驟 3｜設定資料來源連接

逐一設定各資料來源的 API 金鑰或存取路徑。Notion 需要 Integration Token，Google Drive 需要 Service Account。本地檔案直接指定資料夾路徑。

步驟 4｜執行首次資料匯入

啟動資料匯入流程。系統會讀取所有文件、切段、向量化，存入本地向量資料庫。首次匯入視資料量可能需要 5–30 分鐘。

步驟 5｜設定增量同步

配置定時同步（每日或即時），讓新增或修改的文件自動更新到知識庫。避免知識庫和原始資料脫節。

步驟 6｜測試查詢效果

用自然語言問幾個你知道答案的問題，測試查詢準確度。例如：「上次和客戶 A 開會討論了什麼？」「我們的退貨政策是什麼？」

步驟 7｜微調檢索參數

根據測試結果調整：切段大小（chunk size）、相似度閾值、回傳數量（top-k）。通常 chunk size 500–1000 字元、top-k 3–5 是好的起點。

步驟 8｜建立查詢快捷方式

設定 CLI alias 或 Telegram Bot 介面，讓你能隨時快速查詢。例如：`brain "去年Q4營收報告重點"` 就能得到答案。

常見錯誤

  • **一次匯入太多低品質資料**：先從最重要、最常用的 100 份文件開始。垃圾進、垃圾出——低品質文件會污染搜尋結果。
  • **chunk size 設太大或太小**：太大（>2000 字）會包含太多無關內容；太小（<200 字）會喪失上下文。
  • **忘記定期更新**：知識庫建好後就忘了更新，三個月後資料就過時了。務必設定自動同步。
  • **只匯入不整理**：匯入前先清理資料來源：刪除過時筆記、合併重複文件、統一命名規則。

升級方向

  • 團隊知識庫：讓整個團隊共享同一個知識庫，搭配權限控制
  • 自動筆記整理：新增的筆記自動分類、打標籤、建立關聯
  • 智慧寫作助手：寫文章時自動從知識庫中拉出相關資料作為參考
  • 語音查詢：用語音提問，AI 語音回答（適合開車或做家事時使用）
